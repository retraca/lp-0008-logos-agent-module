## Step 1: Standalone keystore + key derivation tests (runs NOW, no special toolchain)

These tests depend only on `aes-gcm`, `argon2`, `rand`, `base64`, `sha2`, `serde_json`, `base58`.

```bash
cd /path/to/lez-wallet-module/lez-wallet-core
cargo test -- --nocapture
```

Expected output: all tests in `src/keystore.rs` and `src/keys.rs` pass. Should complete in < 10s.

If you see a linker error for libargon2, ensure `libclang-dev` (Linux) or Xcode CLI tools (macOS)
are installed. No other system libraries are needed.

## Step 7: Build lez-wallet-core (full, with lez-bridge feature)

PREREQUISITE: steps 3 + 4 + lez-build workspace at ../../lez-build/ relative to the crate.

```bash
cd /path/to/lez-wallet-module/lez-wallet-core
cargo build --release --features lez-bridge
# Output: target/release/liblez_wallet_core.dylib (macOS) or .so (Linux)
# Also: target/release/liblez_wallet_core.a (static)
```

If the build times out (> 2 minutes) it is almost certainly the Risc0 circuit compilation.
That is expected on first build and on clean caches. Let it run to completion; subsequent
incremental builds are fast.

## Step 8: Generate the FFI header (cbindgen)

```bash
cargo install cbindgen  # once
cd /path/to/lez-wallet-module
cbindgen --crate lez-wallet-core \
         --config lez-wallet-core/cbindgen.toml \
         --output qt-module/lez_wallet_ffi.h
# Verify: qt-module/lez_wallet_ffi.h exists and contains lez_wallet_ensure_account etc.
```

cbindgen.toml (create alongside Cargo.toml):

```toml
language = "C"
include_guard = "LEZ_WALLET_FFI_H"
autogen_warning = "/* Generated by cbindgen. Do not edit. */"
```

## Step 11: Install and load in logoscore

```bash
# Build logoscore:
nix build 'github:logos-co/logos-logoscore-cli' --out-link ./logoscore-bin

# Package the module:
mkdir -p modules/lez_wallet_module
cp build/lez_wallet_module_plugin.so modules/lez_wallet_module/
cp metadata.json modules/lez_wallet_module/

# Run a single method call (dev mode, no passphrase security here):
export LEZ_WALLET_HOME=/tmp/lez-agent-test
export LEZ_WALLET_PASSPHRASE=dev-only-passphrase
./logoscore-bin/bin/logoscore \
  -m ./modules \
  -l lez_wallet_module \
  -c "lez_wallet_module.ensure_account(${LEZ_WALLET_PASSPHRASE})" \
  --quit-on-finish
# Expected: prints the agent's AccountId in base58.

./logoscore-bin/bin/logoscore \
  -m ./modules \
  -l lez_wallet_module \
  -c "lez_wallet_module.balance(${LEZ_WALLET_PASSPHRASE})" \
  --quit-on-finish
# Expected: "0" (newly created account with no funds).
```

## CI (integration test against standalone sequencer)

Intended to run as a GitHub Actions job with a self-hosted runner (Nix + Risc0 installed):

```yaml
# .github/workflows/lez-wallet-module.yml
name: lez-wallet-module
on: [push, pull_request]

jobs:
  keystore-unit-tests:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: dtolnay/rust-toolchain@stable
      - run: |
          cd lez-wallet-module/lez-wallet-core
          cargo test -- --nocapture

  integration:
    runs-on: [self-hosted, nix, risc0]
    steps:
      - uses: actions/checkout@v4
      - name: Start standalone LEZ sequencer
        run: |
          cd lez-build
          RISC0_DEV_MODE=1 cargo run -p scaffold -- setup
          RISC0_DEV_MODE=1 cargo run -p scaffold -- start-sequencer &
          sleep 10  # wait for sequencer to be ready
      - name: Build lez-wallet-core
        run: |
          cd lez-wallet-module/lez-wallet-core
          cargo build --release --features lez-bridge
      - name: Build Qt module
        run: |
          cd lez-wallet-module/qt-module
          nix develop --command bash -c "
            cmake -B build -GNinja -DLEZ_WALLET_CORE_DIR=$(pwd)/../lez-wallet-core/target/release
            ninja -C build
          "
      - name: Integration smoke test
        run: |
          export LEZ_WALLET_HOME=/tmp/lez-agent-ci
          export LEZ_WALLET_PASSPHRASE=ci-test-passphrase
          logoscore -m modules -l lez_wallet_module \
            -c "lez_wallet_module.ensure_account(${LEZ_WALLET_PASSPHRASE})" \
            --quit-on-finish
```
